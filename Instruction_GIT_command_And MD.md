# *Язык разметки Markdown*
 Язык разметки Markdown -  это правила для создания структуры и визуального оформления текста с помощью специальных символов — элементов разметки.

 ## Оформление текста
 Что бы оформить текст на языке разметки Marcdown необходимо запомнить некоторые правила приведенные ниже:

 Что бы сделать текст полужирным необходимо его обрамить (**) с двух сторон.

 Например:

 **Аниме на аве, здоровья маме.**

 Или альтернативный способ обрамить (__), с двух сторон.

 Например:

 __Аниме на аве, здоровья маме.__

Что бы сделать текст курсивом необходимо обрамить (*) с двух сторон.


Например:

*Аниме на аве, здоровья маме.*

или альтернативный способ обрамить его (_), с двух сторон.

Например:

_Аниме на аве, здоровья маме._

Что бы текст был одновременно и полужирным и курсивом необходимо скомбинировать основной способ полужирного и альтернативный способ курсива или наоборот.

Например:

__*Аниме на аве, здоровья маме.*__

Что бы сделать текст зачеркнутым необходимо обрамить (~~) с двух сторон.

Например:

~~Аниме на аве, здоровья маме.~~

# *Команды Git*
## Статус работы

Для того что бы посмотреть статус текущей работы воспользуемся командой:
> git status - которая отображает статус работы (сохранен ли , закомичен ли, добавлилн ли и т.д)
покажу пример на текущем состоянии моей работы 
![](status.png)

после введения в терминал команды git status появмлась такая ситуация как на рисунке выше, стрелочка 1 показывает, что файл модифицирован, но при этом не добавлен в остлежививание, стрелочка 2 намекает на подсказку со стороны git что необходимо сделать дальше.
## Добавление файла для отслеживания

Для того что бы добавить файл в ослеживаемые необходимо воспользоваться командой:
> git add name_file - где name_file, имя нашего файла, имя файла можно ввести вручную или же воспользоваться автозаполнением нажав клавишу TAB (над caps lock) посмотрим на результат.

![](gitadd.png)
Надпись сменила цвет с карсного на зеленый, что означает что мы добавили файл в остлеживаемое. Далее необходимо будет сделать "commit"
## Создание фиксации "Commit"

Для того что бы создать коммит "commit" воспользуемся командой:
> git commit -m "message" , где message - текст сообщения которое мы вводим для пометок себе что мы добавили в этом коммите, рассмотрим на примере введем в терминал команду 

> git commit -m "(added heading (Commit)" и нажмем Enter
 
Теперь введем уже известную нам команду git status  и посмотри что нам скажет git 
![](commit.png)

Указатель 1 нам показывает что мы сейчас находимся на ветке "Status", но про ветвление git поговорим не много позже.

Указатель 2 нам говорит, что нечего коммитеть, рабочее древо чистое. В таком виде все сохранено и переживать не о чем).
## Журнал git 

Для того что бы вызвать журнал необходимо воспользоватся командой:
> git log 

Введем эту команду в терминал и посмотрим что он нам скажет:

![](Log.jpg)

Указатель 1 нам показывает номер коммита (commit).
Указатель 2 показывает что сейчас мы находимся в ветке "status" и этот коммит был создан в ней.
Указатель 3 говорит кто создал этот коммит
Указатель 4 показывает время и дату создания коммита.
Указатель 5 указывает на message - сообщение которые мы ввели при команде git commit -m "massage".

Помимо стандартной команды git log, есть еще и команда которая визуализирует ветке и коммиты в журнале и пишется она так: 

> git log --graph 

Давайте пропишем её в терминале и посмотрим что из этого выйдет
